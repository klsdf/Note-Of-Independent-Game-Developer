# 星星之火可以燎原

## 破圈者——Ludum Dare

星星之火可以燎原，第0届GameJam之后，GameJam的浪潮就开始翻涌起来了。

我们刚才说到IGJ0爆火，但这些都仅限于参加GDC以及使用Gamasutra的资深开发者。还没有破圈到一般的开发者群体。

2002年4月（IGJ0后仅一个月） Indie Game Jam的余温尚未散去，一个名叫Geoff Howland的开发者就在自己的网站上发起了一个新的挑战。他受IGJ0启发，但做了一个关键改变：**不再需要线下聚集，不需要昂贵的引擎，任何人都可以在家参与。

这个GameJam规则也很简单：主题是"守卫"，独自一人，一个周末，围绕主题做一款游戏。

最终，有**20个人**响应了这个比赛，他们在IRC聊天室里商量出了GameJam的几个规则：

- 单人开发，不能组队
- 24小时限时
- 从零开始，不能使用现成代码


最终，这场实验性的挑战诞生了**18个作品**。 参与者之一**Mike Kasprzak**后来回忆道："我们做到了，我们很爱这件事，虽然也犯了不少错误，但我们有着屡败屡战、乐于改错的精神。"
没有人想到，这个只有20多人参加的线上比赛，后来会成为**全球规模最大、历史最悠久的线上GameJam**，累计催生了数万款游戏，孕育了《超级食肉男孩》《火线迈阿密》等经典作品的早期原型。至此，大名鼎鼎的**Ludum Dare**（全球最大线上 GameJam）诞生。


![[独立游戏小课堂/GameJam是怎么样风靡全球的？【游戏史小课堂】/asserts/image-4.png]]

当然，24小时单人硬核开发的门槛还是太高了。后来Ludum Dare逐渐演化出两个组别，适应不同人群：
- **Compo组**：延续硬核传统，**48小时**、单人开发、必须提供代码，所有内容必须从零创作
- **Jam组**：更加休闲，允许团队参赛，时间延长至**72小时**，可以有限度地使用现成资源和代码



## 规范制定者——Nordic GameJam


如果说Ludum Dare让GameJam"破了圈"，那么2006 年 1 月的Nordic Game Jam则为后来的全球化扩张制定了真正的可复制模板。

在丹麦哥本哈根IT大学的教室里，一群学者和开发者策划了第一届Nordic Game Jam。

核心人物之一**Jesper Juul**（哥本哈根IT大学游戏学者）注意到一个问题：Indie Game Jam的核心是**单人程序员+技术实验**，强调技术突破，门槛其实很高。他和Gorm Lai（资深游戏程序员、《LIMBO》开发者之一）、Henriette Moos（哥本哈根IT大学外部讲师）商量：
能不能做一个**真正面向所有人**、**强调团队协作**、**不那么硬核**的游戏开发活动？
几个哥们一拍即合，设计了一套后来成为行业标准的规则：

1. **团队协作**：鼓励组队，而不是单人作战。这样就可以狠狠抱大佬大腿！
2. **非竞赛性质**：强调交流和实验，而不是比拼谁的游戏更好 
3. **线下聚集**：必须面对面，强调人与人之间的化学反应 
4. **48小时**：确定了一个"刚好能做完又不至于累死"的时间长度

在Jesper Juul保存的2006年原始策划文档中，他们这样描述这个活动：

> “Nordic Game Jam是一个在周末举办的研讨会，主题是‘用周末做一个游戏’，探讨游戏设计和技术问题，并结识其他从事游戏设计和开发的人。”


这套模式被证明极其有效。Nordic Game Jam也成为大学将GameJam引入学术教育的先驱，让游戏开发从"极客的业余爱好"变成了"可以研究的对象"。而这些规则，也后世成为了大部分GameJam的“村规”。

Jesper Juul后来回顾时特意提到，这种“以团队为中心”的模式，与Indie Game Jam那种“以程序员+引擎为中心”**的模式形成了关键区别。正是这种模式，让完全不懂技术的人（美术、设计、音乐、策划）也能参与进来，真正实现了“游戏开发属于所有人”

## 天啊，是Global GameJam大人！

有了Nordic Game Jam的成熟模板，几位行业先驱决定做一次更大胆的实验：**把这场狂欢推向全世界**。

GGJ的故事始于2008年瑞典的Gotland游戏奖颁奖典礼上。
当时，Susan Gold（IGDA，也就是国际游戏开发者协会）教育特别兴趣小组的主席在现场遇到了**Gorm Lai**（Nordic Game Jam的联合创始人之一，上文我们刚介绍完他）。

Susan向Gorm提出了一个在当时看来近乎疯狂的想法：**为什么不把Game Jam推向全世界？让不同国家、不同时区的开发者在同一个周末、围绕同一个主题做游戏？

Gorm最初有些怀疑，但很快，Nordic GameJam团队同意成为这场全球实验的旗舰站点。Susan接着动用自己在IGDA的人脉四处游说，**Ian Schreiber**（游戏设计教材《Challenges for Game Designers》的合著者）很快加入，成为第三位联合创始人。

由于Nordic GameJam已经是一个成熟的年度活动，有固定的举办日期，他们只有短短四个月的时间来筹划这场全球性的实验。时间紧迫，但想法已经点燃，伟大的时刻即将到来。

2009年1月30日下午5点（GMT+13时区），首届Global Game Jam在（GGJ）新西兰达尼丁率先拉开帷幕。随后，全球各地的站点逐时区点亮。
最终，首届Global Game Jam在**23个国家、53个站点**同步举行，吸引了超过1600名开发者参与，创造出了370款游戏。
一夜之间，GameJam不再是硅谷极客或北欧学生的专利——从东京到圣保罗，从赫尔辛基到墨尔本，世界各地的开发者都在同一个周末、围绕同一个主题、用48小时创造属于自己的游戏。

这场活动也被开发者简称为GGJ。

# GameJam前传

我们已经把GameJam的主线内容讲完了。但是我们要注意，Gamejam的诞生绝不是偶然，而是历史的必然。（总感觉这句话有点熟悉）
类似于牛顿发现了万有引力定律一样，在他之前有哈雷、开普勒等科学家已经做了很多铺垫了。

我们已经知道了GameJam起源于0th Indie Game Jam，那么0th Indie Game Jam又是起源于哪里呢？受到了哪些文化影响呢？

现在观点也是众说纷纭，不过我比较认可下面的几个观点：

1.  **即兴协作的“果酱”文化**：“Jam”一词借自爵士乐的“即兴演奏”，指乐手们无排练地临时合奏。GameJam完美移植了这种精神：限时、即兴、跨角色协作，共同完成一次创作。
2. **2001年GDC上的"Dogma 2001"挑战**：2001年GDC上的“Dogma 2001”挑战直接启发了后来的Indie Game Jam创始人。他们最初想挑战商业束缚，但随后反其道而行之，决定**利用新技术进行创造**，这才有了2002年的第0届Indie Game Jam。
3. **技术限制下的创意爆发**：在GameJam正式诞生前，开发者就常受限于早期硬件（如超低像素、极小容量），这种“戴着镣铐跳舞”的经历，孕育了在限制中追求创意的精神。一个著名的精神先例是，1992年John Carmack（天啊，是约翰卡马克大人！）在飞行途中用几小时做出了《德军总部3D》原型，体现了“极限开发”的雏形。也影响了后世的千千万游戏开发者。
4. **技术界的“黑客马拉松”传统**：黑客马拉松（Hackathon，国内一般叫黑客松）这个词本身是黑客（Hack）和马拉松（Marathon）的合成词。它的核心概念很简单：让一群极客在限定时间内（通常是24-48小时），通宵达旦地编程，完成一个软件项目。这也是GameJam的公认的文化来源。



等等！你是不是看到了熟悉的词？Dogma 2001！我们刚才好像提到过？！

没错，Dogma 2001是游戏设计师**Ernest Adams**于2001年发布的一份**挑战性宣言**，旨在对抗当时游戏行业过度依赖技术、忽视创意本质的趋势。它的灵感直接来源于1995年丹麦导演拉斯·冯·提尔等人发起的电影运动**Dogme 95**。

当时，这两位导演对好莱坞电影过度依赖特效、炫技拍摄手法的风气深感不满。他们认为，电影的本质应该是"演员在故事中扮演真实的人"，而不是被技术包装的空壳。于是他们起草了一份被称为"守节誓言"的十条戒律（这里就不细讲了）。

Ernest Adams受到启发，在Gamasutra上发表了一篇题为 《Dogma 2001: A Challenge to Game Designers》 的文章，矛头直指那些信奉"技术至上"的开发者。Adams在文中提出了一个鲜明的论点：技术扼杀创意。他观察到，每当新一代游戏硬件问世，游戏行业的整体创意水平就会明显下降。主要原因在于：开发者把过多精力放在熟悉新硬件上，大家都忙着思考如何利用新技术，而不是如何创造新玩法。结果就是大量“换皮游戏”充斥市场。

于是Adams这份宣言也设计了10条限制性规则，强迫开发者把注意力从"如何利用新硬件"转移到"如何创造好游戏"上。由于内容太长了，之后有机会给大伙细说。这里简单展示其中最有趣的3条吧。
- **禁用硬件3D加速**：游戏必须在640x480的16位色模式下以20帧/秒运行，旨在让开发者回归玩法本身，而非炫技
- **限制陈词滥调题材**：彻底禁止出现"骑士、精灵、矮人、龙"以及"没完没了的吸血鬼"等幻想题材，以此迫使开发者探索新鲜的故事世界
- **禁止特定游戏类型**：明确排除第一人称射击、横版卷轴、现代军事模拟等类型，鼓励创造真正意义上的新类型

虽然这些规则有点抽象，但它成功引发了一场关于"技术vs创意"的行业大讨论，也直接启发了2002年的IGJ0.

你是否发现了？IGJ0和Dogma 2001的观点完全相反！Dogma 2001认为是技术的发展限制了开发者的创意，而IGJ0则相反，积极使用新技术（他们自研的精灵渲染引擎）证明了技术不仅不会限制想象力，反而能激发更多的创意。

因此IGJ0才会写文章：《技术启迪创意：Indie Game Jam 颠覆了 Dogma 2001》来反驳Dogma 2001的观点。

没有Unity和UE的普及，GGJ从1600人增长到3.5万人，是不可想象的。

俗话说，学哲学就是学哲学史，实际上学游戏也是在学游戏史。我自己参加GameJam的时候，经常会遇到得奖心切，争一保二的朋友。
但看完这段历史你就会发现：GameJam从来就不是比赛。
从IGJ0的好哥们集会，到Ludum Dare的论坛，到Nordic Game Jam的教室，再到GGJ的全球狂欢——GameJam的本质一直是**认识朋友、找灵感、尝试新技术**。
希望各位在看完文章后，能以更平和的心态参加GameJam，大家一起开开心心做游戏多好啊！一味地去搞窝里斗，到底还是不太好的嘛。
