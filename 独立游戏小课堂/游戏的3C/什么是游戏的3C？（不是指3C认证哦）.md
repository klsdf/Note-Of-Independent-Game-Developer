# 3C的提出

游戏的 **3C** 是游戏设计的核心基础概念之一，这一概念最早是由法国游戏公司**育碧（Ubisoft）** 提出，所谓的3C指 **Character（角色）、Camera（相机 / 镜头）、Control（控制）**，三者共同决定玩家最直接的操作手感与沉浸体验。
作为一家开发了《刺客信条》、《细胞分裂》等众多知名3D动作游戏的大厂，育碧很擅长解决这种“**角色如何动、玩家怎么看、操作怎么反馈**”的问题。因此，育碧才能在长期的实战经验中总结出3C理论。

之后，2013年，资深游戏设计师Scott Rogers在其著作《通关！游戏设计之道》（_Level Up! The Guide to Great Video Game Design_）中，对这一概念进行了更详细的阐释和拓展，使其开始在更广泛的游戏开发者群体中传播，逐渐成为行业内的通用术语。

# 3C详解

## Character（角色）

角色是玩家在游戏中的 “化身”，决定玩家能做什么、怎么做。
角色是最关键的部分，因为游戏大部分时间都能看到角色，角色的手感和观感直接影响了游戏的体验
- 一般来说，角色都是包含模型和动画的，但是这些内容更多会交给美术来处理。比如角色的样貌，衣着打扮之类的。一个小萝莉站立的时候，手就更适合轻轻翘起。也更适合穿美味的白丝，因为白丝会给人清纯、圣洁，可以直接开舔的感受。尤其是小小的、香香的、软软的、糯糯的小脚，不敢想有多美味，美味！！！！！美味！！！！！！！
- 除此之外，策划还要定义角色的碰撞体大小和形状、角色质量、重力系数、最大速度、加速度/减速度、摩擦系数等。这些决定了角色的手感和物理感（注意Control部分也可以影响手感、手感有多个影响部分，之后会细说）。

## Camera（相机 / 镜头）

相机是玩家观察游戏世界的 “眼睛”，决定玩家看到什么、怎么看。
这里的Camera不仅仅是视角的问题，它还包含了很多运镜问题：
- 比如在开发2D俯视角游戏的时候，如果让摄像机完全跟随玩家移动，就会导致运镜很僵硬。这时如果给摄像机跟随加一个阻尼效果，就会让画面舒适很多。Unity内部也内置了Cinemachine来实现各种复杂的运镜。
- 如果是3D游戏，当玩家进入建筑物内部后，还得考虑到底是把摄像机拉近防止穿墙，还是把墙面做半透明显示。
- 一些二游为了防止玩家偷看裙底，往往会控制摄像机在倾斜角过大时，透明角色的身体或者抬升镜头。但是也有很多玩家利用角色的滞空状态来制造神秘打野点。

## Control（控制）

控制是连接玩家输入与游戏响应的桥梁，决定操作的手感。
这里的Control更接近交互的层面，比如说：
- 在Xbox的手柄映射时，使用x轻击，y重击或许是不错的设计。但是如果设计Y轻击，A重击，B确认，X跳跃则会非常折磨。
- 如果在开发任天堂游戏的时候，因为老任的手柄经常手柄漂移（然而我的原版和oled都没有漂移，运气很好），所以设定**摇杆死区**也非常重要。而《塞尔达》就没有摇杆死区，所以网上也可以看到林克自动跑路的视频。
- 在FTG类游戏时，因为广泛存在硬直的设计，但是玩家又很难做到在硬直结束后立刻输入，这时可以将玩家的下一次输入缓存一小段时间（例如0.2秒）。这样就能解决了“按键没反应”的挫败感，是动作游戏“手感丝滑”的关键技术。
- FPS中Control更是重量级，当年Bungie的《光环》在上主机之后，为了解决手柄难以操作的问题，他们设计当准星划过敌人Hitbox时，灵敏度会自动降低（减速区域），帮助玩家微调瞄准；以及举枪瞄准时准星自动轻微吸附到最近的敌人身上。一举解决了手柄手感太差的问题。
这些都是在设计Control时需要考虑的问题。



# 3C的适用条件

3C的理论也不是在所有游戏都适用的，比如说Galgame、俄罗斯方块、水果忍者这种无明确的角色操控的游戏，就无法使用3C理论来分析。我们也把一些3C特征明显的游戏叫做**重3C游戏**。

最后总结一下3C游戏的特征吧：
1. **存在一个可连续控制的角色化身**（非单位群控）；
2. **摄像机与角色之间存在动态空间关系**（非固定俯视/横版锁定）；
3. **输入是连续的模拟/数字信号**（摇杆、鼠标、键盘方向），而非离散的点选；
4. **手感是核心体验支柱**——即玩家会反复通过操控来获得反馈与乐趣。
